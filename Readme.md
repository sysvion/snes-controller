March 15
---
Update: From what you may already have read, I designed (and printed) a case for the SNES controller port. And all the holes of the controller are placed. But i did it using openscad because i kinda rage quit trying to install and use [Autodesk Fusion](https://www.autodesk.com/eu/products/fusion-360/overview). My primary operating system is linux and for years i use a VM without GPU passthrough to run windows applications. But CAD software needs a lot of VRAM that i do not have. So i looked at alternatives that run on linux and i found freecad and openscad. [FreeCAD](www.freecad.org) works the same as autodesk. The draw, constraint and extrude workflow, which almost all CAD software uses. And if the UI is designed well, I think it is a pretty good workflow. But at last all the UIs I tried are very picky, and if you make the slightest of errors clicking when you are constraining a drawing you must go back to the other shape that you want to constrain. For a cad noob it is not nice to use. Also where do i need to look if i did it correctly? And where are the warnings? - i went back to the language based way of designing shapes using [OpenSCAD](https://openscad.org/). And well it the way of handling reminds me of how [Jetpack Compose](https://developer.android.com/compose) works but it uses its own language. The system is very comforting. But that is probably because i'm used to that workflow. And i need to learn new methods if i want to stay up to date and do a lot of things. A lecturer gave me a remark that it is better to use industry standard software because others can easely change the models that i'm making. But you can go by using OpenSCAD. The remark was after i have shown my printed design and as far as the minor is concerned, the project is done. But i do continue because it is a nice project. And i want to controll my snes.

During the next weekend I spend some time testing the electrical engineering part of Autodesk fusion on a old windows 10 system. And it looks like what i was expecting. It was nice that the 3d part reacts to a change of the pcb size. And to have [SPICE](https://en.wikipedia.org/wiki/SPICE) integration is also great. But i first need to create an electrical connection between the snes and a PCB which im going to design. I do not know the terms but i think i need a crimp contact or a bus contact. And i have bought a connection that i thought it will fit. But at last it did not. Now i am returning the product by post. But all this takes a lot of time.
